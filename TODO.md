## Идея: Один универсальный tool — выполнение TS-кода

Возможно, одного инструмента достаточно — выполнить TypeScript/JavaScript код.

Все текущие инструменты (`read`, `write`, `bash`, `grep`, `find`, `ls`, etc.) — это по сути функции на TS/JS. Если бы был один инструмент `execute`, который запускает произвольный код в контролируемой среде, он мог бы заменить всё остальное.

### Плюсы
- Минимализм — один инструмент вместо множества
- Гибкость — можно делать всё, что позволяет язык
- Не нужно помнить схемы параметров для каждого инструмента

### Минусы
- Безопасность — нужна изолированная sandbox (как у Code Interpreter)
- Предсказуемость — LLM лучше генерирует JSON, чем произвольный код
- Обработка ошибок — сложнее парсить произвольные exceptions
- Доступ к внешним сервисам — SQLite, API keys, subagent-инфраструктура
- Производительность — генерация кода требует больше токенов
